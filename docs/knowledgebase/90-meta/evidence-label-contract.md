# Labels

- `[USER]` user-provided requirements or style fixtures.
- `[REPO]` directly observed repository code, config, scripts, or tests.
- `[OFFICIAL]` vendor/platform documentation.
- `[DISCORD_EXPORT]` evidence from exported Discord history.
- `[SYNTH]` reasoned synthesis from multiple labeled inputs.
- `[OPEN]` unresolved questions, caveats, or unknowns.

# Rules

- [SYNTH] Every non-trivial claim in knowledgebase docs should include one label.
- [SYNTH] Do not mix factual repo observations and inferred implications in one unlabeled sentence.
- [SYNTH] High-impact recommendations should not rely on one source alone.
- [SYNTH] Date-sensitive findings should include recency context.
